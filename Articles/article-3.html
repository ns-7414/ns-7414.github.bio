就跟文章標題一樣，這個網站也是上架到Github後運作的。
原本一開始還在煩惱要怎麼弄出網域，甚至還要在幾乎不理解網站背後的運作下嵌入圖片等等，後來正好想起看過有人的網站網址名稱是用github結尾的，就順手找了一下

又正好想起先前看過的一句話「能教別人才是真正的學會」，決定在學會用Github架網站的同時自己也來寫一下教學。

創建Github帳號

在 Grid 出現之前，要做複雜的二維佈局（既要控制橫向，也要控制縱向）， 通常要靠 float、position，或者 Flexbox 巢狀嵌套。這些方法都有缺點： float 很難控制、position 脫離文件流、Flexbox 本質上是一維的。

Grid 的核心概念是：把容器切成格子，然後告訴每個子元素要佔哪些格子。 聽起來很簡單，但理解這個心智模型之後，很多以前覺得麻煩的佈局問題都消失了。

最基本的三個屬性

① grid-template-columns

定義欄的數量與寬度，是 Grid 裡最常用的屬性。

/* 三欄等寬 */
.container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 16px;
}
// 三欄等寬示意

fr 是 fraction 的縮寫，意思是「剩餘空間的幾份」。 repeat(3, 1fr) 就是把容器的寬度平分成三等份。 如果你想要左側固定、右側自動填滿，可以寫 200px 1fr。

② auto-fill 與 auto-fit

這是我踩過最多次坑的地方。這兩個關鍵字都可以搭配 minmax() 使用， 讓 Grid 自動決定要放幾欄，實現響應式佈局。 

/* 自動填滿，每格最窄 120px */
.auto-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(120px, 1fr));
  gap: 8px;
}
// auto-fill — 根據容器寬度自動計算欄數

關鍵字 空欄格的處理方式 適用情境
auto-fill 保留空格（格子存在，但是空的） 元素數量不固定，需要維持格線對齊
auto-fit 折疊空格（現有元素自動拉伸填滿） 少量元素，希望它們撐滿整排

③ grid-column 與 grid-row（讓元素跨格）

這是 Grid 最強大的功能之一，可以讓特定元素橫跨多個欄位或多個列。 

/* 讓第一個子元素佔滿兩欄 */
.feature-card {
  grid-column: span 2;   /* 或寫 1 / 3 */
  grid-row: span 2;      /* 同時跨兩列 */
}

Grid 最容易踩到的坑

Grid 是作用在「容器」和「直接子元素」之間的關係。 孫元素不會自動參與 Grid 佈局，除非你也對它的父元素設定 display: grid。 很多奇怪的排版問題，都是因為搞錯了「誰是 Grid 容器」。
WARN 在 Grid 容器內，子元素的 float 會失效。 同樣地，display: inline-block 也沒有作用。 這是刻意的設計，但如果你在 Grid 裡還試著用 float 排版，會讓你很困惑。
TIP 瀏覽器開發者工具（DevTools）現在對 Grid 的視覺化支援非常好。 在 Chrome 或 Firefox 裡選取 Grid 容器，旁邊會出現一個 Grid 的 badge， 點下去就會在頁面上顯示格線，是學 Grid 最好用的工具。

這個網站怎麼用 Grid

這個作品集網站的主要佈局都是用 Grid 做的。 主頁的卡片區塊是 grid-template-columns: 1fr 1.15fr， 讓右側的技能卡片稍微寬一點。文章頁面的正文加側邊欄是 1fr 200px，在手機上則用 media query 切換成單欄。

最有趣的是作品集頁面的圖片格子，用了 repeat(3, 1fr) 加上 統一的 aspect-ratio: 4/3，不管圖片本身的尺寸是多少， 都能保持整齊的格狀排列。這個組合在做作品集類型的網頁時非常好用。
